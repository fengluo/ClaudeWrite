# 🗄️ Archive - 归档

归档是已完成或不再活跃的内容的存放地。

## 什么时候归档？

以下情况应该归档:
- ✅ 项目已完成
- ✅ 想法已放弃或不再相关
- ✅ 资源过时或不再使用
- ✅ 领域不再维护

## 归档而非删除

归档的好处:
- 保留历史记录
- 可以回顾学习
- 可能将来有用
- 减少主工作区的干扰

## 归档组织

保持原有的结构:
```
04_Archive/
├── 2024/
│   ├── Projects/
│   │   └── Completed-Project/
│   ├── Areas/
│   └── Resources/
└── 2023/
    └── ...
```

或按日期扁平化:
```
04_Archive/
├── 2024-01-Project-Name/
├── 2024-02-Old-Idea/
└── ...
```

## 归档清单

归档项目前:
- [ ] 更新项目状态为 "completed"
- [ ] 整理和清理笔记
- [ ] 提取可复用的知识到资源或领域
- [ ] 添加归档日期
- [ ] 移动到归档目录

## 搜索归档

归档的内容仍然可以搜索:
- 使用 VSCode 的全局搜索 (`Cmd+Shift+F`)
- 使用 Claude 的 Grep 工具
- 使用 `npm run stats` 包含归档统计

## 定期回顾

建议每季度或每年:
1. 回顾归档内容
2. 提取有价值的洞察
3. 删除确实不再需要的内容
4. 考虑重启某些想法

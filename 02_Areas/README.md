# 📂 Areas - 领域

领域是需要持续维护的责任区域，没有明确的截止日期。

## 什么是领域？

领域的特征:
- ✅ 持续性的责任或兴趣
- ✅ 需要定期维护和更新
- ✅ 没有"完成"的概念
- ✅ 包含相关的标准和流程

## 与项目的区别

| 项目 | 领域 |
|------|------|
| 有截止日期 | 持续进行 |
| 可以完成和归档 | 持续维护 |
| 短期目标 | 长期责任 |
| 例: 写一篇文章 | 例: 健康管理 |

## 常见领域示例

- **职业发展** - 技能学习、职业规划
- **健康** - 运动、饮食、健康记录
- **财务** - 预算、投资、理财
- **家庭** - 家庭事务、亲子教育
- **写作** - 写作习惯、风格指南
- **学习** - 持续学习的主题
- **工作** - 工作相关的流程和标准

## 领域文件结构

每个领域可以包含:
```
02_Areas/
└── Area-Name/
    ├── README.md       # 领域概述和标准
    ├── notes.md        # 持续更新的笔记
    └── resources.md    # 相关资源链接
```

或者使用单个文件:
```
02_Areas/
└── area-name.md        # 简单领域用单文件
```

## 使用模板

复制 `_template/area-template.md` 创建新领域:
1. 复制模板文件
2. 重命名为领域名称
3. 填写内容
4. 定期更新

## 定期回顾

建议每周或每月回顾领域:
- 检查是否符合标准
- 更新相关信息
- 识别需要启动的新项目

# Daily Review - 每日回顾

自动统计当天活动，引导反思，记录进展和洞察。

## 工作流程

### 1. 扫描今日活动

使用 Bash 工具查找今天修改的文件：

```bash
# 查找今天创建的文件
find . -type f -name "*.md" -newermt "today" ! -path "./node_modules/*" ! -path "./.git/*"

# 查找今天修改的文件
find . -type f -name "*.md" -newermt "today" ! -newermt "tomorrow" ! -path "./node_modules/*"
```

统计并分类：
- 新增笔记
- 更新笔记
- 所属文件夹分布

### 2. 生成活动摘要

格式化展示今日活动：

```
📊 今日活动统计

## 文件变化
- 新增笔记: 3 篇
- 更新笔记: 5 篇
- 总字数变化: +1,250 字

## 活跃项目
在以下项目中工作：
1. [[React-Hooks-Tutorial]] - 更新了 3 个文件
   - draft.md (新增 800 字)
   - notes.md (添加实践经验)
   - references.md (新增 2 个参考资料)

2. [[API-Design-Guide]] - 更新了 1 个文件
   - README.md (完善目标描述)

## 活跃领域
02_Areas/
- [[Writing]] - 添加了新的写作技巧

## 新增资源
03_Resources/
- Articles/react-best-practices.md (新保存的文章)

## 收件箱
00_Inbox/
- 新增 2 个想法待处理
```

### 3. 识别主题模式

分析今日内容，提取核心主题：

**方法**：
- 提取所有标签和关键词
- 统计频率
- 识别主题集群

**展示**：
```
## 今日核心主题

基于你今天的活动，识别出以下主题：

1. **React 开发** (6 次提及)
   - 相关文件: [[Tutorial]], [[Notes]], [[Resources]]
   - 主要关注: Hooks, 最佳实践, 性能优化

2. **API 设计** (3 次提及)
   - 相关文件: [[API-Guide]]
   - 主要关注: RESTful, 一致性

3. **写作技巧** (2 次提及)
   - 相关文件: [[Writing-Area]]
   - 主要关注: 技术写作, 结构化表达
```

### 4. 引导性反思问题

向用户提出 5 个反思问题：

```
## 反思时刻

请花几分钟回顾今天：

### 1. 今日亮点
今天最有价值的洞察或成果是什么？
（例如：完成了某个重要任务、理解了某个概念、解决了困扰已久的问题）

> [等待用户输入]

### 2. 完成任务
今天完成了哪些重要任务？
（列出具体的、可衡量的成果）

> [等待用户输入]

### 3. 挑战与困惑
遇到了什么挑战或困惑？
（记录问题比假装没有问题更有价值）

> [等待用户输入]

### 4. 新知识
学到了什么新知识或技能？
（再小的收获也值得记录）

> [等待用户输入]

### 5. 明天计划
明天需要继续或开始的工作是什么？
（3-5 个具体任务）

> [等待用户输入]
```

## 行为准则

### 必须做 ✓

- ✓ 自动统计，减少用户手动输入
- ✓ 提出有深度的反思问题
- ✓ 保存完整记录，包括统计和反思
- ✓ 建立今日活动与已有笔记的连接
- ✓ 提供积极的反馈和建议

### 禁止做 ✗

- ✗ 不要问太多问题（5个足够）
- ✗ 不要替用户填写反思内容
- ✗ 不要忽略"没有进展"的日子（记录同样重要）
- ✗ 不要批评或评判用户的工作

## 高级功能

### 识别趋势

如果连续多天回顾，识别模式：
```
📈 趋势观察：
- 过去 7 天持续关注 "React" 主题
- "API设计" 项目活跃度上升
- 收件箱积压增加，建议运行 /inbox-processor
```

### 庆祝里程碑

识别特殊成果：
```
🎉 里程碑！
今天完成了 [[Project-X]] 的初稿，
这是你第 [N] 天连续工作在这个项目上。
```

### 关联洞察

如果今天的主题与之前的洞察相关：
```
💡 相关洞察：
今天关于 [主题] 的工作让我想起你在 [[Insight-2024-01-05]]
中提到的观点。考虑更新那篇洞察？
```

## 最佳实践建议

向用户建议：

1. **固定时间** - 每天同一时间进行回顾（如晚上工作结束时）
2. **诚实记录** - 包括挑战和失败，而非只记录成功
3. **保持简短** - 5-10分钟完成
4. **定期回顾** - 每周或每月回顾每日记录，发现模式

## 记住

每日回顾不是绩效考核，而是：
- **觉察** 自己的工作模式
- **记录** 思考的演变过程
- **调整** 明天的计划
- **积累** 长期的进步

最好的回顾是让用户感到清晰和掌控，而非压力和焦虑。
